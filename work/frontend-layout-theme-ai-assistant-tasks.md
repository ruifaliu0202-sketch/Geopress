1. 建立前端产品化布局与视觉主题基线
   定义新工作台的全局体验目标：左侧可折叠菜单、左中右三段窗口结构、顶部留白快捷入口、奶油白与鼠尾草绿品牌主题、系统组件立体背景。这个任务的产出应成为后续页面改造的设计约束，避免每个页面单独发散。范围限定在前端视觉与交互基线，不改变业务 API 和后端模型。

2. 将主导航迁移为窗口左侧可折叠菜单
   把当前顶部导航承载的主菜单能力调整为左侧常驻导航，并支持展开、折叠、当前视图高亮和移动端可用状态。这个任务是整体布局的前置依赖，因为后续三段式内容区和顶部快捷区都需要释放当前 AppBar 的空间。需保持现有 `ViewKey`、引导定位点和平台后台入口不丢失。

3. 重构应用外壳为左中右三段式窗口框架
   在左侧菜单稳定后，将工作区主界面整理成左侧导航、中部主内容、右侧上下文/辅助区域的产品框架。这个任务解决当前页面内容全部堆在单一主容器里的问题，为 AI 悬浮工作台、快捷入口、上下文辅助信息预留稳定空间。需要保证已有概览、知识库、媒体矩阵、战役、达人合作、技能包、合规报告等页面在新框架下仍可完整显示。

4. 建立顶部留白快捷入口区
   在窗口顶部预留轻量快捷操作区域，承载工作区选择、刷新、引导程序、账号状态和常用入口。这个任务让高频操作从页面内容中抽离出来，减少主内容区干扰。需要与左侧菜单、移动端布局和现有 onboarding tour 协同，不应遮挡业务页面标题和表单。

5. 将工作区工作台改造成可插拔 AI 助手悬浮组件
   把当前工作台面板从页面内容内迁移为悬浮式 AI 助手入口，并抽象为可替换的组件契约。首个形象使用卡通柯基犬，所有新生成视觉资产必须使用 image2 模型，并保留后续替换为其他 IP 形象的能力。该任务只承载入口、状态、动作和形象组件化，不把具体业务页面逻辑耦合进助手内部。

6. 产品化 AI 助手动作与工作台快捷能力
   在悬浮 AI 助手稳定后，将生成内容、创建知识库、绑定账号、创建计划、打开引导等高频工作台动作接入助手面板。这个任务让工作台从静态快捷卡片升级为更像 AI 助手的操作中心。需要保证动作仍复用现有 dialog、view 和 reload 机制，避免产生第二套业务流程。

7. 建立可复用的按钮动效与 IP 高亮组件
   抽象产品级按钮动效能力，包括重要/IP 按钮的循环高亮扫光动画、hover/active 反馈和低干扰的状态变化。这个任务把动画从一次性样式提升为可复用组件，方便后续技能包、AI 助手、VIP、引导等入口统一使用。需要兼顾可访问性和低性能设备，不能让循环动画影响表单操作和页面阅读。

8. 用奶油白与鼠尾草绿重塑 MUI 主题和系统组件立体感
   基于新的品牌主题调整 MUI palette、背景层级、卡片、纸面、按钮、Chip、Dialog、Drawer、Table 和 Section 的默认观感。这个任务解决当前界面偏通用后台感的问题，通过亮度差、边框、阴影和柔和背景建立轻量立体层次。范围应覆盖共享组件和主题层，避免每个业务页面重复写零散样式。

9. 适配现有业务页面到新框架与新主题
   将已有工作台页面逐一调整到新布局、主题和组件规范下，确保内容密度、表格宽度、卡片间距、空状态、错误状态和移动端表现一致。这个任务是跨页面收口，目标是让此前新增的媒体矩阵、战役、达人合作、技能包、合规报告页面与老页面形成统一产品。需要保留所有既有业务入口和 API 行为，不做无关业务重构。

10. 完成视觉回归、响应式验收和交互收口
   对新布局在桌面、窄屏、移动端下做完整验收，重点检查左侧折叠菜单、顶部快捷区、右侧区域、AI 悬浮助手、弹窗、引导遮罩和循环动效是否遮挡或溢出。这个任务用于消除布局改造带来的体验风险，并把 lint/build 等验证纳入交付标准。范围是前端体验收口，不新增新的产品功能。
